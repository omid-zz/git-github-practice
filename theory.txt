پاسخ سوال 1:
Git یه برنامه‌ست که روی کامپیوتر نصب می‌شه و کمک می‌کنه تغییرات فایل‌ها رو ذخیره کنیم و به عقب برگردیم.
GitHub یه سایت هست که می‌تونیم پروژه‌هامون رو اونجا آپلود کنیم تا بقیه هم ببینن یا با هم همکاری کنیم.
یعنی Git برای کنترل نسخه‌هاست و GitHub برای اشتراک‌گذاری و همکاری روی پروژه‌ها.

پاسخ سوال 2:
هر دو برای ترکیب تغییرات استفاده می‌شن.
Merge تغییرات رو به صورت جداگانه نگه می‌داره و تاریخچه رو حفظ می‌کنه.
ولی Rebase تغییرات رو پشت سر هم مرتب می‌کنه تا تاریخچه تمیزتر بشه.
برای پروژه‌های گروهی معمولاً Merge امن‌تره.

پاسخ سوال 3:
وقتی دو نفر همزمان یه فایل رو تغییر بدن و بخوان با هم ترکیب کنن، Git نمی‌دونه کدوم نسخه رو نگه داره و conflict ایجاد می‌شه.
برای حلش باید فایل رو باز کنیم، قسمت‌هایی که با هم فرق دارن رو ببینیم و تصمیم بگیریم کدوم رو نگه داریم.
بعد از اصلاح، باید تغییرات رو ذخیره و commit کنیم.

پاسخ سوال چهارم:
local repository پروژه ای است که ما برای سیستم خود داریم و ان را با کمک گیت کنترل میکنیم
remote repository نسخه ای از پروژه است که روی گیت هاب یا بطور کلی در اینترنت وجود دارد
ما این امکان را داریم که تغییرات را از لوکال به ریموت بفرستیم یا تغییرات را ریموت دریافت کنیم

پاسخ سوال پنجم:
این دستور ما را از وضعیت کلی پروژه مطلع میکند
به عنوان مثال نشان میدهد کدام فایل تغییر کرده اند 
یا اینکه کدام فایل ها اماده کامیت کردن هستند یا چیزی برای کامیت نداریم

پاسخ سوال ششم:
چون این برنچ نسخه نهایی ما در پروژه خواهد بود و در صورت اشتباه کردن امکان خراب شدن پروژه وجود دارد
در نتیجه باید هر فردی که در پروژه شرکت میکند برنچ حود را بسازد و تغییرت را در ان برنچ اعمال کند
در نهایت برنچ خود را در برنچ اصلی مرج کند تا نسخه اصلی پروژه به مشکل نخورد
